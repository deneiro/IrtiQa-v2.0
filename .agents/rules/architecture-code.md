Единый источник истины (Single Source of Truth): Все константы (математические формулы расчета уровней, HEX-коды 8 цветов атрибутов, цены на предметы в магазине) хранятся строго в одном конфигурационном файле (например, src/shared/config/game-core.ts). Запрещено хардкодить цвета или формулы внутри компонентов.

Optimistic UI (Оптимистичный интерфейс): Для базовых действий (отметить привычку, выполнить задачу) интерфейс должен реагировать мгновенно, не дожидаясь ответа от базы данных Supabase. Начисляем XP и Золото на экране сразу, базу обновляем в фоновом режиме.

Feature-Sliced Design (FSD): Строгое соблюдение изоляции модулей. Логика Маркета не должна напрямую зависеть от логики Журнала. Все взаимодействия между модулями происходят на уровне глобального стора (Zustand) или страниц.

Безопасность данных (RLS): Вся информация пользователя (особенно Social Hub, Долги и Журнал) является строго конфиденциальной. Доступ к строкам базы данных возможен только для авторизованного владельца (user_id = auth.uid()). Выполнение прямых SQL-запросов с фронтенда в обход политик безопасности запрещено.
