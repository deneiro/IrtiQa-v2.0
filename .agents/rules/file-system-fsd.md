Этот проект строго следует методологии Feature-Sliced Design (FSD), адаптированной под Next.js App Router. Агентам категорически запрещено нарушать иерархию слоев и правила именования.

1. Глобальная структура монорепозитория
Разделение зон ответственности между агентами начинается с корневых папок:

📁 supabase/ — Территория Бэкенд-агента (Cloudopus). Здесь лежат migrations/ (SQL-скрипты) и functions/ (Edge Functions). Фронтенд-агент сюда не лезет.

📁 src/ — Территория Фронтенд-агента (Gemini). Весь React/Next.js код. Бэкенд-агент может писать сюда только в один файл: src/shared/types/database.types.ts.

📁 public/ — Статические ассеты (звуки, иконки, базовые изображения).

2. Иерархия слоев FSD (внутри папки src/)
Агенты должны располагать код строго по этим слоям (снизу вверх, от Shared к App). Верхние слои могут импортировать нижние, но никогда наоборот.

🧠 shared/ (Самый нижний слой): Переиспользуемые UI-компоненты (кнопки, инпуты), глобальные типы (база данных), конфиги (цвета атрибутов), утилиты (date-fns). Не знает ничего о бизнес-логике.

🧬 entities/ (Сущности): Базовые бизнес-сущности. Например, User, Quest, Habit. Здесь лежат только их интерфейсы, сторы (Zustand для хранения) и UI-карточки без сложной логики.

⚙️ features/ (Фичи): Пользовательские сценарии и действия. Например, habit-check-in (кнопка выполнения привычки), auth-by-email, create-smart-quest. Фича может использовать несколько сущностей.

🧱 widgets/ (Виджеты): Самостоятельные блоки интерфейса, собранные из фич и сущностей. Например, UserProfileCard, HabitsList, Sidebar.

📄 views/ (Страницы): Композиция виджетов. Собирают целые страницы.

🚀 app/ (Фреймворк): Настройки Next.js (App Router). Файлы layout.tsx, page.tsx (которые просто импортируют готовую страницу из слоев views/), глобальные стили и провайдеры.

3. Правило Public API (Критично для ИИ-агентов)
Каждый модуль (slice) в папках features, entities или widgets обязан иметь файл index.ts в корне.

Правило: Импортировать что-либо из другого модуля можно ТОЛЬКО через его index.ts.

❌ Неправильно: import { HabitCard } from '@/entities/habit/ui/HabitCard'

✅ Правильно: import { HabitCard } from '@/entities/habit' (экспорт настроен внутри index.ts).

4. Правила именования файлов и папок
Папки: Всегда kebab-case в нижнем регистре (например, create-quest, social-hub).

React-компоненты: Всегда PascalCase (например, UserProfile.tsx, HabitTracker.tsx). Расширение строго .tsx.

Утилиты, хуки, сторы: Всегда camelCase (например, useHabitStats.ts, questStore.ts, formatDate.ts).

Стили: Используем Tailwind прямо в className. Никаких .module.css или .scss файлов не создаем, чтобы не засорять проект.

5. Изоляция Zustand-сторов
Глобальное состояние не должно лежать в одном огромном файле store.ts.

Сторы разбиваются по сущностям. У Привычек свой стор (entities/habit/model/store.ts), у Профиля свой (entities/profile/model/store.ts).
